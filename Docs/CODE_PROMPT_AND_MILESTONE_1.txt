PART 3 — EXEMPLAR SELECTION PROMPT (for collecting voice samples)
═══════════════════════════════════════════════════════════════

WHAT THIS IS:
A prompt to use in a FRESH Claude chat (or Claude Project) to have
Claude help you select the best 10-15 voice exemplars from your
archive of past articles. You do this ONCE per client.

HOW TO USE:
1. Open a new Claude chat (Claude.ai) — does NOT need to be in a
   project. Fresh slate is fine.
2. Upload 15-25 of your best past articles for the client as .txt files
   or paste them inline (whichever is easier).
3. Paste the prompt below, replacing [CLIENT NAME] with the actual
   client name (iTech Valet, Brighton Gold, etc.)
4. Review Claude's output. Swap any selections you disagree with.
5. Save the final formatted output as voice_exemplars.txt
6. Upload to the client's 4.3B Drive folder
7. Add the file ID to Client Manager (folderVoiceExemplars field)


═══ PASTE THIS PROMPT IN A FRESH CLAUDE CHAT ═══

I'm building a content production pipeline that uses your past
articles as voice exemplars to teach an AI to write in my voice.

CONTEXT:
I am [YOUR NAME], operator of [CLIENT NAME]. My writing style is:
- Direct, casual, conversational ("beer with a friend" energy)
- Strong POV from the first sentence
- Zero hedging language
- Heavy use of conversational transitions (But, And, So, Here's, That means)
- Hard data preserved with specificity (real numbers, not "many" or "several")
- Clear thesis defended, not summarized
- Slightly confrontational when calling out competitors or industry BS
- Short paragraphs, varied length, with rhetorical questions for rhythm

The articles I'm uploading were all written or heavily edited by me.
Some are stronger than others. I need you to pick the ones that BEST
represent my voice and have the strongest POV.

INTENT LAYER TAXONOMY:
I write articles across 5 intent layers. Sort the selected exemplars
into these buckets:

1. DIRECT INTENT — The reader knows what they want and is ready to act
   (e.g., "Best [service] in [location]")
2. INDIRECT INTENT — The reader has a problem your service solves but
   hasn't named the service (e.g., "How to fix [problem]")
3. LATENT INTENT — The reader has an underlying issue they don't know
   is connected to your service (e.g., "Why does X happen?")
4. COUNTER-INTENT — The reader is evaluating objections, myths, risks
   (e.g., "Is [service] safe?" or "Is [service] worth it?")
5. POST-INTENT — The reader has already used the service and wants
   maintenance or next steps (e.g., "What to do after [service]")

YOUR JOB:
1. Read every article I uploaded.
2. Select 10-15 total that BEST represent my voice — strong POV, zero
   hedging, conversational rhythm, hard data, clear thesis.
3. Sort selections across the 5 intent layers (aim for 2-3 per intent
   if possible — but quality over distribution; if I gave you 7
   Indirect Intent articles and only 1 Direct Intent, that's fine).
4. For each selected article, write a 1-2 sentence "Why this works"
   note explaining what makes it strong (the friction setup, the
   callback, the conviction moment, etc.)

OUTPUT FORMAT (exact format, copy verbatim):

=== DIRECT INTENT EXEMPLAR 1 ===
Title: [article title]
Why this works: [1-2 sentences on what makes the voice strong]

[full article text]

=== DIRECT INTENT EXEMPLAR 2 ===
Title: [article title]
Why this works: [1-2 sentences]

[full article text]

=== INDIRECT INTENT EXEMPLAR 1 ===
[and so on for all 10-15 selections]

CRITICAL FORMATTING RULES:
- Use the exact === [INTENT] EXEMPLAR N === marker format. JS parses
  these markers. Any variation breaks the parser.
- Include the FULL article text for each exemplar, not summaries.
- Number sequentially WITHIN each intent layer (Direct 1, Direct 2,
  not globally).
- If an intent layer has zero good exemplars from my uploads, skip it.
  Don't fabricate or stretch to fill it.

Begin the analysis. Show me your reasoning for the top picks BEFORE
producing the final formatted file.

═══ END OF PROMPT ═══

AEO Article Creation Pipeline at Phase 5 (Voice Draft).

File: voice_exemplars.txt
File ID: [enter Drive file ID after creating the file]

Structure inside the file uses these markers (parsed by JS):
  === DIRECT INTENT EXEMPLAR N ===
  === INDIRECT INTENT EXEMPLAR N ===
  === LATENT INTENT EXEMPLAR N ===
  === COUNTER-INTENT EXEMPLAR N ===
  === POST-INTENT EXEMPLAR N ===

The pipeline reads the Intent Layer column from the 4.2 sheet row
(populated by 4.1 V4.0) and loads only the matching-intent exemplars
into Phase 5's cached context. This ensures Claude sees concrete
examples of voice in the exact intent layer being written.

If voice_exemplars.txt is missing or has no exemplars for the target
intent, the pipeline falls back to closest intent (Direct↔Indirect,
Latent→Indirect, Counter→Indirect, Post→Latent), then to no exemplars
(voice rules only). This is handled by Error Recovery code F12.
