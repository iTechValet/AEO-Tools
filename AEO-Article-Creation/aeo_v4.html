Phase 2 — V1 in TWO passes: 2A Structure Frame → 2B Data Hydration
2A (submit_v1_frame): structural skeleton only — H2/H3 headings, table HEADERS (no data), FAQ QUESTIONS (no answers), link anchors mapped to the research brief, image slots, CTA placement, checklist, thesis. No prose, no numbers. JS validates the frame, then 2B (submit_v1_hydration): paragraph prose, FAQ answers, and table DATA ROWS — every row { cells, sourceReferenceId } citing a Phase 1 factClaim; rows with no supporting factClaim are dropped, not invented. JS merges 2B into the frame keyed by node id, so structure physically cannot drift. Phase 3 failures are repaired by surgical micro-calls (submit_v1_corrections) that touch only the failing nodes — the V1 is never regenerated. Shared cached context across 2A / 2B / fix.
